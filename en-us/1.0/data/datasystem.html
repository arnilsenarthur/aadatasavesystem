DataSystem

DataSystem is the main namespace of package. You can use and modify it however you want to meet your needs in the best possible way

The system was developed mainly with the following objectives:

  • General data storage
  • Save System
  • Packet / Multiplayer messaging
  • Faster as possible
  • Less storage usage as possible
  • Secure / Non-Human easily readable

Package Contents

The main package has a set of source code scripts and a folder with an example script for you to test the main features. Use the test example source to learn and to pratice.

Main Concepts

Data

The object DataSystem.Data is the main object data of package. It holds the entire data hierarchy and is used to convert your data to bytes and vice versa. To create a new Data you can just use:

using DataSystem;
...
Data data = new Data(); //Create data

Debug.Log(data.ToString()); //Prints data

DataItem

The object DataSystem.DataItem is the base for all objects stored in Data. There are 3 main data-object types:

  • DataGroup Holds a dictionary of data-objects (With string as keys)
  • DataList Holds a list of data-objects
  • DataValue Holds a value of a defined type (You must register custom types)

DataConverter

Data converter is used to convert data to bytes and bytes to data. You may use it to customize security keys or choose how your data will be serialized or readed. The package have 3 default data converters:

  • DataConverter Default converter. Uses GZip to compress, accepts security keys and is SYNC, without callbacks
  • UncompressedDataConverter No compress data, ideal for high speed sending/receiving of small/medium data, like game packets. Works SYNC, without callbacks
  • AsyncDataConverter ASYNC converter. Ideal for game conversions, while game wont freeze while converting data. With callbacks that are fired when conversion ends. You can mix AsyncDataConverter with the other converters (See data conversion section)

Data Creation / Modification

Data data = new Data(); //Create data

//C# Types
data.Put("justANumber",13);
data.Put("myName","Árnilsen Arthur");

//Unity types
data.Put("greatColor",Color.yellow);
data.Put("aVector",new Vector3(0,1.2f,0.5f));

//Child Groups
DataGroup gp = new DataGroup();
gp.Put("x",25.24);
gp.Put("y",7.22);

data.Put("position",gp);

Debug.Log(data.ToString()); //Prints data
//{justANumber=13,myName="Árnilsen Arthur",greatColor=color(1,0,9215686,0,01568628,1),aVector=vector3(0,1,2,0,5),position={x=25,24d,y=7,22d}}

In the example above we created a data and stored data inside it. You can modify values too, or remove if you want:

//Getting values
string name = data.GetString("myName");
double pX = data.GetGroup("position").GetDouble("x");

//Remove value
data.Remove("position");

//Clear group or list
data.Clear();

//Or you can just replace a value using put again:
data.Put("greatColor",Color.red);

//You can use the generic put and get DataItem methods, but if you create a new data type you must to register it before
DataString st = new DataString("Testing data system");
data.Put("generic",st);

//Replace value directly
((DataString) data.Get("generic")).value = "New string value";

A Data object is a DataGroup, so you can use all methods from datagroup in a data object, like Clear(), Put(..), etc… A DataGroup object acts like an table where objects are referenced by a key:

But if you only want to store a list of values without keys, just by order, you can use DataList. A data list acts like an normal List, where you can Add, Put, Get, Remove or Clear values:

DataList list = new DataList();
list.Add(new Vector2(0f,1f));
list.Add(new Vector2(0f,-1f));
list.Add(new Vector2(1f,0f));
list.Add(new Vector2(1f,0f));
Debug.Log(list.Count()); //4

//Remove item
list.Remove(3);

//Insert in position 
list.Put(1,"Teste");

You can mix various object types in same list. You can insert DataGroups in DataLists and DataLists in DataGroups. (Cyclic insertion is not checked, so caution)

Data Conversion

Data conversion allows a Data object to be transformed into bytes to be sent via network or saved to files and etc., and also allows the contender to occur: bytes to be transformed into Data.

Bytes are a group of 8 bits (0 and 1) that represents a number from 0 to 255, like 01001001 = 73. The system is optmized to use less storage as possible even without compression. Because of this we have a key format limitation. The only characters accepted on a DataGroup key is 0-9 A-Z a-z and _

To convert Data <-> bytes we can use this default simple way:

Data data = new Data();
...
byte[] bytes = data.ToBytes();

Data newData = Data.Read(bytes);

In the example above we are converting SYNC using compression but not protection. This means that if the data is very large, the game will freeze until operations are completed, and as there are no protection keys, the data is easier to be changed by third parties

To use protection keys we need to use a new argument,passing true to this methods:

Data data = new Data();
...
byte[] bytes = data.ToBytes(true);

Data newData = Data.Read(bytes,true);

The only problem now is that the system will use the same predefined keys, so we can use the method Using(a,b,c) to change the keys that will be used now:

Data data = new Data();
...
Data.Using(13,66,93); //Set your keys here
byte[] bytes = data.ToBytes(true);

Data newData = Data.Read(bytes,true);

This means that if the keys configured at the time of reading are not identical to the time of writing, all the content cannot be read:

Data data = new Data();
...
Data.Using(13,66,93); //Set your keys here
byte[] bytes = data.ToBytes(true);

Data newData = Data.Read(bytes,true); //Will read entire data

Data.Using(12,63,91); 
Data newData = Data.Read(bytes,true); //Will read a empty object {}

But theres another problem: Concurrency. Imagine you are trying to convert two objects with different security keys at the same time. The system will you mix things up. Thats why we recommend you to use non-static DataConverter:

DataConverter a = new DataConverter();
DataConverter b = new DataConverter();

a.Using(24,99,128); //Set different keys
b.Using(22,189,43); //Set different keys

Data data = new Data();
...

byte[] bytesA = a.ToBytes(data,true); //Using converter A 
byte[] bytesB = b.ToBytes(data,true); //Using converter B

Data readRight = a.Read(bytesA,true); //Will works because key is the same
Data readWrong = a.Read(bytesB,true); //Won`t work because keys used to create bytesB array are different than a keys.

But even it is ideal to work with concurrency it will not deal with game-freezing problems. To solve this you must use AsyncDataConverter

AsyncDataConverter cv = new AsyncDataConverter();

Data data = new Data();
...

cv.ToBytesAsync(data,false,(bytes) => {
    //Converted async!
    cv.Read(bytes,false,(newData) => {
        //Converted async!
    });
});
//It will execute before conversions end

This way we can do conversions without freezing the game. Using callbacks you can now exactly when a conversion is done. There are pros and cons in all the two converter types, but theres an other converter, the UncompressedDataConverter. As said in the name, it will not compress/decompress data, but it is faster than the other two methods. But you can mix Async and Uncompressed converters as writen bellow:

AsyncDataConverter cv = new AsyncDataConverter(new UncompressedDataConverter());
//Same way

Default Data Types

As default, data system supports various data types (C# pure data types and unity data types):

You can use event .Put(value) or .Put(new DataString(value)) (For all default types, excluding DataNull and DataBoolean). All default alias has .Put .Get and .Add alias (Excluding DataString256 and DataByteArray256).

Custom Data Types

Follow the following steps to add custom data types support:

Caution while creating custom data types, or you will can corrupt your Data bytes.
But you can add support to your custom data types, or just for another types that dont have default support. To create support for a new data type you can follow the steps as shown bellow: 
//First create a class for your custom type, like the shown bellow (Look for DataSystem.cs for examples)
public class DataByte : DataValue<byte> //byte represents that the stored value will be a byte
{
    //Main constructor used to build object
    public DataByte(byte value) : base(value)
    {

    }


    //Empty constructor to provide a base object while deserializing
    public DataByte() : base(0)
    {

    }

    public override byte GetCode()
    {
        return 1; //Most important value: Must be equals the index of type in BUILDER array + 1. The DataByte is in the index 0 of the array, so the code must be 1.
    }
    public override int Deserialize(int i, byte[] bt)
    {
        //Here you create a deserialization method for your data
        value = bt[i];
        return i + 1; //Here you must return i (current index) + amount of used bytes, in this case 1 (in types with variable size like string you will must return the exactly size)
    }

    public override void Serialize(DataOutput data)
    {
        //Here you must to write all current data to bytes
        data.WriteByte(value);
    }

    public override string ToString()
    {
        //Just for visual debug, when you use Data.ToString() for example
        return value + "b";
    }
}

//Now you need to add a builder provider to deserialization:
//Go to file DataSystem.cs in class DataItem in the variable BUILDERS (Line 149) and add the default provider, following the code order. This represents the default BUILDERS variable:

public static Func<DataItem>[] BUILDERS = new Func<DataItem>[]
{
    () => new DataByte(), //Code: 1
    () => new DataInt(), //Code: 2
    () => new DataString(), //Code: 3
    () => new DataFloat(), //Code: 4
    () => new DataDouble(), //Code: 5
    () => new DataString256(), //Code: 6
    () => new DataByteArray(), //Code: 7
    () => new DataByteArray256(), //Code: 8
    () => FALSE, //Code: 9
    () => TRUE, //Code: 10
    () => NULL, //Code: 11
    () => new DataColor(), //Code: 12
    () => new DataVector2(), //Code: 13
    () => new DataVector3(), //Code: 14
    () => new DataDateTime(), //Code: 15

}; //Add custom types here

//After this you will be able to use your custom data format!
