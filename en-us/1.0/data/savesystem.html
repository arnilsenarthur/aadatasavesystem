Save System

SaveSystem is a section of package developed based in the DataSystem to help developers to create a complete save system for small, medium and large games. Using SaveSystem you can create fast system that dont freezes game while saving or read data. All the data will be compressed to use less storage space. All functions that need some times to process data are async, so game wont freeze while system is writing or reading things. Because of this you need to use Callbacks. The SaveSystem supports two diferent structures:

  • Single files saves
  • Folder saves (1 folder per save)

Creating Save System

using SaveSystem;

//Create new SaveStructure of single files
SaveStructure structure = new SaveStructure(path,SaveStructureType.FILES);

//Create a new save ASYNC in save structure
structure.CreateSave("New Save 1",(save) =>{
    //Save created and default files are generated
    //Get main save file
    save.GetSaveFile((saveFile) => {
        //Get data from file
        Data data = saveFile.GetData();

        //Fill data
        data.Put("justATestData",14.25f);

        //Save data to file
        data.Save();
    });
});

...

//Get a save by name
Save save = structure.GetSave("New Save 1");

//Get save file
save.GetSaveFile((saveFile) => {

    //Get a data from save file
    float myNumber = data.GetData().GetFloat("justATestData");
});

The example above shows how to create a new save of FILES type, where on save have only one file. So if you want to retrieve the SaveFile you must use .GetSaveFile().

Creating Folder Save System

using SaveSystem;

//Create new SaveStructure of folders
SaveStructure structure = new SaveStructure(path,SaveStructureType.FOLDERS);

structure.CreateSave("New Save 1",(save) => {
    //Save created and files are generated using current file provider (See providers section)

    //In a `FOLDERS` save you must specify what file you want to get or create (subfolder support, use /, like dialogs/core)

    //Create default data
    Data data = new Data();
    data.PutString("testName","�rnilsen Arthur");

    //Create file with default data
    save.CreateSaveFile("test",data,(saveFile) => {
        //File test.data created with default data
    });
});

...

//Get save file (1st way - SYNC)
//If file is not loaded yet it will return null, but it will starts to load file async:
Save save = structure.GetSave("New Save 1");

SaveFile file = GetSaveFileOnlyLoaded("test"); //will return null because file is not loaded
//Maybe after few millseconds file will be available (Depends on data load time)

//Get save file (2nd way - ASYNC)
Save save = structure.GetSave("New Save 1");
GetSaveFile("test",(saveFile) => {
    //Callback called after file is loaded
    string name = saveFile.GetData().GetString("testName");
});

Callbacks

Since the save system has operations that involve reading and writing files, some functions had to be written asynchronously so that your games do not crash while the system performs these operations. If these methods were synchronized, the player could experience crashes. Because of this we use the system of callbacks that return the values you need:

//Calling this function and waiting response in callback
GetSaveFile(file,callback<SaveFile>)

//Never pass a null callback. Pass an empty callback like this
GetSaveFile(file,(saveFile) => {});

Providers

Providers are used to auto generate default data in saves, making the things easier:

SaveStructure structure = new SaveStructure(SaveStructureType.FILES);

//Set a provider for single file saves (all saves will contain this data. You can customize it for each save, like the value saveName)
structure.WithSingleFileProvider((save) => {
    Data data = new Data();
    data.Put("defaultInt",4);
    data.Put("saveName",save.GetName());
    return data;
});

//In file providers you can set what files must be generate by default with custom data
structure.WithFolderPopulator((save) => {
    Data data = new Data();
    data.Put("name", save.GetName());
    data.Put("creation_date", DateTime.Now);

    //Create info file
    save.CreateSaveFile("info", data,(s) => { });
});

By default FILES saves will be empty and FOLDERS saves will use SaveStructure.DefaultSaveFolderPopulator as main populator. If you change the populator you can call SaveStructure.DefaultSaveFolderPopulator to populate folder as default too.

structure.WithFolderPopulator((save) => {
    //Do custom things

    //Call default provider
    SaveStructure.DefaultSaveFolderPopulator(save);
});

Protection

The SaveSystem suports the protection key system. You can set your custom keys and enable/disable protection if you want:

SaveStructure structure = new SaveStructure(SaveStructureType.FILES);

//Enable protection
structure.WithProtectionKeys(12,56,78);
//DO THINGS

//Disable protection
structure.WithoutProtectionKeys();
